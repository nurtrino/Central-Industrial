# Monkey Read Monkey Do

> **⚠️ LOCAL-ONLY — not deployed to Render.** This tool needs a CUDA GPU, and its
> whole point is that audio never leaves your machine. It runs on your local GPU box
> and is part of this repo only for completeness. The Central Industrial hub links to
> it as a `[ LOCAL ]` tool via the `HUB_URL_MONKEYREADMONKEYDO` URL (e.g. a Tailscale
> address). See the repo [README](../README.md) for the overall architecture.

Local audio/video transcription + meeting-notes engine.

- **Transcription & diarization run 100% locally** (Whisper `large-v3` + pyannote on the RTX 5090). Audio never leaves this machine.
- **Notes** are generated by Claude from the **text transcript only** (no audio transits the network), following the house style guide in `transcipt and summarize notes.md`.

## One-time setup (already done)

The GPU stack lives in `.venv` (Python 3.12, CUDA 12.8 PyTorch). To rebuild: `bash setup_env.sh`.

## Add your two credentials

Edit **`.env`**:

- `HF_TOKEN` — free HuggingFace token for diarization. Create an account, accept the licenses at
  `huggingface.co/pyannote/speaker-diarization-3.1` and `huggingface.co/pyannote/segmentation-3.0`,
  then make a read token at `huggingface.co/settings/tokens`.
- `ANTHROPIC_API_KEY` — for the notes step.

Without `HF_TOKEN` transcription still works but won't label speakers. Without `ANTHROPIC_API_KEY` only transcribe-only (Option 1) works.

Optional in `.env`:
- `NOTEMAX_OFFLINE=1` — forces the local audio stage (Whisper + diarization) **fully offline**: zero HuggingFace network calls while audio is processed (models must be cached once first; they already are). Belt-and-suspenders privacy. The badge bar shows `offline` when on.
- `NOTEMAX_WHISPER_MODEL` (default `large-v3`), `NOTEMAX_MODEL` (notes model, default `claude-opus-4-8`).

## GPU

Whisper and pyannote both load into **GPU VRAM** (float16). Monkey Read Monkey Do refuses to run if no CUDA GPU is visible rather than silently falling back to CPU/system memory. Each run logs the GPU name and live VRAM usage.

## Visual exhibits

Drop `.png`/`.jpg` screenshots or slides alongside the audio (Options 2 & 3). During note creation Claude reviews each image and places it (as `[[EXHIBIT n]]`) at the most contextually relevant point per the style guide; any image it can't place is appended at the end. Exhibits render inline in the browser window **and** are embedded in `notes.docx`. Genuine data tables shown in a slide are also reproduced as Word tables. Note: images are sent to the Claude API for placement — only **audio** is held strictly local.

## Run

Double-click **`Monkey Read Monkey Do.vbs`**. This one launcher:
- starts the server **only if it isn't already running** (idempotent — never spawns a duplicate),
- runs it **hidden** (no console window; logs still go to `server.log`),
- opens the page in your browser.

It's the single entry point — open it any time; if Monkey Read Monkey Do is already up it just opens the page. (A web page can't start its own server, so this launcher is the trigger.)

Prefer a console (to watch logs / Ctrl-C to stop)? Run directly:

```
.venv/Scripts/python.exe app.py
```

### Always-on (optional)
To have Monkey Read Monkey Do available the instant you open the bookmark, put a shortcut to `Monkey Read Monkey Do.vbs` in your Startup folder (Win+R → `shell:startup`) so it launches at login. Ask and I can set this up.

Then open http://127.0.0.1:5005

A **⟳ Restart server** button sits in the lower-left of the page — it spins up a fresh process (clearing in-memory jobs) and the page reconnects automatically in ~1s. (Logs continue appending to `server.log`; after a web restart the server runs detached, so use Task Manager to stop it, or just close it from the original console if you launched it there.)

**Mascot:** save a picture as `mascot.jpg` (or `.png`/`.jpeg`/`.webp`) in the Monkey Read Monkey Do folder and it shows in the middle of the page. If no such file exists the spot is simply hidden.

## Two-pass notes (editorial review)

For Options 2 & 3 the notes go through **two** Claude passes:
1. **Draft** — the transcript is turned into notes per `transcipt and summarize notes.md`.
2. **Editorial review** — a fresh senior-reviewer pass per `notes_editor_review.md` checks the draft against the transcript (and any images) for completeness, translational/number errors, style, and image placement, then returns the corrected final notes plus a changelog. The final reviewed notes are what's saved/shown; the changelog is saved as `editor_changelog.txt` and shown under "Editor changes" in the results.

## Preferences

The **⚙ Preferences** button (top-right) opens an in-page editor for the two summarization prompts:
- `transcipt and summarize notes.md` — the **first-pass** notes style guide
- `notes_editor_review.md` — the **second-pass** editorial review prompt

Edit either, click **Save**, and the new version is used on your **next task** (the prompts are re-read from disk on every job). A `.bak` of the prior version is kept beside the file. **Revert** restores the last saved version.

## New Task

After a job finishes (or is stopped), **＋ New Task** copies that job's deliverables (transcript `.txt`, `notes.docx`, `notes.md`, `editor_changelog.txt`) to **`<Desktop>\Monkey Read Monkey Do Output\<date> - <source name>\`**, then clears the app and waits for the next file. (Browsers can't see the original source folder's real path, so the save target is the Desktop output folder, organized by date + source filename; duplicate names get ` (2)` appended.) Everything also remains in `work/<id>/`.

## Pause / Stop

While a job runs, **⏸ Pause** freezes it (resume with the same button) and **⏹ Stop** immediately writes whatever is finished to the work folder (partial transcript, and notes if reached) and stops. Pause/Stop are most responsive during transcription (checked per segment) and during the notes/review streaming; the brief diarization step isn't interruptible mid-call.

## Modes

1. **Transcribe only** — media → diarized transcript.
2. **Transcribe & summarize** — media → transcript → notes + Word doc.
3. **Summary from transcript** — drop an existing `.txt` transcript → notes + Word doc.

Drop one or more audio/video/text files (and optional slide/screenshot images, which are
placed as exhibits in the notes). Output appears in the browser window and is saved under `work/<timestamp>_<id>/`
as `transcript.txt`, `notes.md`, and `notes.docx`.
